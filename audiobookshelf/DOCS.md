# Audiobookshelf — Full Documentation

## Getting Started

### First Launch

1. Start the add-on and click **Open Web UI** to open Audiobookshelf in a new browser tab at `http://<your-ha-ip>:8133/audiobookshelf/`.
2. On first visit you will be prompted to create the **root user** (admin). There is no default username or password.
3. After creating the root user, you can add additional users from **Settings > Users** with custom permissions.

### Adding Libraries

Audiobookshelf organizes content into **libraries**. Each library points to a directory on disk.

1. Upload your files to the appropriate HA directory (e.g., `/media/audiobooks`) via Samba, Filebrowser, or SSH.
2. In the Audiobookshelf web UI, go to **Settings > Libraries > Add Library**.
3. Set the **path** to the container path (e.g., `/media/audiobooks`), not the host path.
4. Choose the library type: **Book** (audiobooks and ebooks) or **Podcast**.
5. Click **Scan** to populate the library.

You can create multiple libraries (e.g., one for audiobooks, one for podcasts) and add multiple paths to a single library.

### Supported Formats

**Audio:**

| Format | Notes |
|--------|-------|
| MP3 | Universally supported. Some seeking issues with very long files. |
| M4B, M4A | Convenient: one file per book with chapter support. |
| FLAC, OGG, OPUS | Supported via ffprobe. Opus works for streaming but Apple devices cannot download for offline playback. |
| Any audio format | If ffprobe can detect it, Audiobookshelf can use it. |

**Ebooks:**

| Format | Progress tracking |
|--------|-------------------|
| EPUB | Full support |
| PDF | Supported |
| CBR, CBZ | Supported |
| AZW3, MOBI | Limited, no progress tracking |

### Directory Structure

**Audiobooks** — Each folder is one book. Recommended structure:

```
Author Name/
  Series Name/
    Vol 1 - 1994 - Book Title {Narrator Name}/
      Audio Track 1.mp3
      Audio Track 2.mp3
      Cover.jpg
    Vol 2 - 1995 - Second Book/
      Audiobook.m4b
  Standalone Title/
    Audio File.m4a
```

Folder naming supports author name, title, publish year, series sequence, subtitle, and narrator (in `{braces}`). Disc/CD folders (`Disc 1`, `CD 2`) are merged automatically.

**Podcasts** — Flat structure:

```
Podcast Title/
  Episode 1.mp3
  Episode 2.mp3
  Cover.jpg
```

### Metadata

Metadata is parsed in this order (configurable per library in **Settings > Scanner**):

1. **Folder structure** — title, authors, narrators, series, sequence, year from folder names
2. **Audio file tags** (ID3) — artist, album, composer mapped to author, title, narrator, etc.
3. **Sidecar files** — `desc.txt` (description), `reader.txt` (narrator), `.opf` files
4. **`metadata.json`** — auto-generated by Audiobookshelf

Online metadata (Audible, Google, OpenLibrary) is **not** fetched during scans. Use the **Match** or **Quick Match** feature on individual items to fetch metadata and covers from online providers.

## Configuration

All options are configured in the Home Assistant add-on **Configuration** tab.

### Library Paths

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `audiobooks_path` | string | `/media/audiobooks` | Directory for audiobook files. Create as a library in the web UI after first launch. |
| `podcasts_path` | string | `/media/podcasts` | Directory for podcast files. Create as a library in the web UI after first launch. |
| `backup_path` | string | _(empty)_ | Custom path for database backups. When empty, backups are stored in the add-on config volume (`/data/metadata/backups`). |

### Podcast

| Option | Type | Default | Env var | Description |
|--------|------|---------|---------|-------------|
| `podcast_download_timeout` | int (1–300) | `30` | `PODCAST_DOWNLOAD_TIMEOUT` | Timeout in seconds to wait for a podcast RSS feed download to start. |
| `max_failed_episode_checks` | int | `24` | `MAX_FAILED_EPISODE_CHECKS` | Maximum consecutive failed episode auto-download checks before auto-download is disabled for a podcast. Set to `0` to never auto-disable. Leave empty to use the upstream default (24). |

### Network

| Option | Type | Default | Env var | Description |
|--------|------|---------|---------|-------------|
| `router_base_path` | string | `/audiobookshelf` | `ROUTER_BASE_PATH` | Base URL path prefix for all routes. Change only if you use a reverse proxy serving Audiobookshelf under a different subpath. The default `/audiobookshelf` works with direct port access. |
| `allow_cors` | bool | `false` | `ALLOW_CORS` | Enable Cross-Origin Resource Sharing. Needed if a separate web app or browser extension accesses the Audiobookshelf API. |
| `exp_proxy_support` | bool | `false` | `EXP_PROXY_SUPPORT` | Enable experimental proxy support. Respects `HTTP_PROXY`/`HTTPS_PROXY` environment variables for outbound requests. **Warning:** This also disables the SSRF request filter. |

### Security

| Option | Type | Default | Env var | Description |
|--------|------|---------|---------|-------------|
| `allow_iframe` | bool | `true` | `ALLOW_IFRAME` | Allow the UI to be embedded in iframes. Enabled by default for HA iframe panels and Dashboard cards. |
| `disable_ssrf_filter` | bool | `false` | `DISABLE_SSRF_REQUEST_FILTER` | Disable Server-Side Request Forgery protection. May be needed if self-hosting podcasts on the same server. |
| `ssrf_whitelist` | string | _(empty)_ | `SSRF_REQUEST_FILTER_WHITELIST` | Comma-separated domain hostnames to exclude from the SSRF filter (e.g., `localhost,internal.example.com`). Only effective when the SSRF filter is enabled. |
| `rate_limit_auth_window` | int | `600000` | `RATE_LIMIT_AUTH_WINDOW` | Rate limiting window in **milliseconds** for authentication endpoints. Default is 10 minutes. Leave empty to use the upstream default. |
| `rate_limit_auth_max` | int | `40` | `RATE_LIMIT_AUTH_MAX` | Maximum authentication attempts per IP within the rate limit window. Set to `0` to disable rate limiting entirely. Leave empty for the upstream default. |
| `access_token_expiry` | int | `3600` | `ACCESS_TOKEN_EXPIRY` | Access token expiration in **seconds**. Default is 1 hour. Shorter values are more secure but require more frequent re-authentication. Leave empty for the upstream default. |
| `refresh_token_expiry` | int | `2592000` | `REFRESH_TOKEN_EXPIRY` | Refresh token expiration in **seconds**. Default is 30 days. Leave empty for the upstream default. |

### Media Delivery

| Option | Type | Default | Env var | Description |
|--------|------|---------|---------|-------------|
| `use_x_accel` | bool | `false` | `USE_X_ACCEL` | Enable Nginx `X-Accel-Redirect` headers for optimized file serving. Enable only if using an Nginx reverse proxy in front of the add-on. |

### Advanced

| Option | Type | Default | Env var | Description |
|--------|------|---------|---------|-------------|
| `skip_binaries_check` | bool | `false` | `SKIP_BINARIES_CHECK` | Skip the automatic ffmpeg/ffprobe binary download and version check at startup. The upstream Docker image already includes these. |
| `query_logging` | string | _(empty)_ | `QUERY_LOGGING` | SQL query logging level. Set to `log` to log all SQL queries, or `benchmark` to log queries with execution times. Leave empty for no logging. Useful for debugging performance issues. |
| `sqlite_cache_size` | int | _(empty)_ | `SQLITE_CACHE_SIZE` | Set the `cache_size` pragma for the SQLite database. Controls the number of database pages held in memory. Higher values use more RAM but can improve query performance. Leave empty for the SQLite default. |
| `sqlite_mmap_size` | int | _(empty)_ | `SQLITE_MMAP_SIZE` | Set the `mmap_size` pragma for the SQLite database. Memory-mapped I/O can improve read performance on supported systems. Leave empty for the SQLite default. |
| `sqlite_temp_store` | int (0–3) | _(empty)_ | `SQLITE_TEMP_STORE` | Set the `temp_store` pragma: `0` or `1` = file-based temp storage, `2` or `3` = memory-based. Leave empty for the SQLite default. |
| `env_vars` | list | `[]` | _(varies)_ | Custom environment variables passed directly to Audiobookshelf. Each entry has a `name` and optional `value`. Useful for setting variables not exposed as dedicated options (e.g., `TZ`, `HTTP_PROXY`). |

### Auto-managed Variables

The following environment variables are set automatically by the add-on and cannot be overridden through configuration:

| Env var | Value | Purpose |
|---------|-------|---------|
| `CONFIG_PATH` | `/data/config` | Database and settings. Must be on local storage. |
| `METADATA_PATH` | `/data/metadata` | Cache, streams, covers, downloads, logs. |
| `PORT` | `8133` | Listening port. Fixed for the add-on. |
| `SOURCE` | `docker` | Installation source identifier. |

## File Access

The add-on maps the following Home Assistant directories with read/write access:

| HA directory | Container path | Typical use |
|-------------|---------------|-------------|
| Add-on config | `/data` | Database, settings, metadata |
| Media | `/media` | Audiobooks, podcasts |
| Share | `/share` | Alternative media storage, backups |

Upload files via the **Samba** add-on, **Filebrowser** add-on, or SSH. Then set the library path in the Audiobookshelf web UI to the **container path** (e.g., `/media/audiobooks`), not the host path.

## Example Configurations

### Basic — audiobooks and podcasts

```yaml
audiobooks_path: /media/audiobooks
podcasts_path: /media/podcasts
```

### Custom media paths with backup

```yaml
audiobooks_path: /share/books/audiobooks
podcasts_path: /share/podcasts
backup_path: /share/backups/audiobookshelf
```

### Secure setup with stricter rate limiting

```yaml
rate_limit_auth_window: 300000
rate_limit_auth_max: 10
access_token_expiry: 1800
```

### Debugging with query logging and SQLite tuning

```yaml
query_logging: benchmark
sqlite_cache_size: -20000
```

### Custom env vars (timezone, proxy)

```yaml
env_vars:
  - name: TZ
    value: Europe/Amsterdam
```

## Adding to the HA Sidebar

Since Audiobookshelf does not support HA ingress, you can add it as a Dashboard iframe panel:

1. Go to **Settings** > **Dashboards**
2. Click **Add Panel** > **Webpage**
3. Set the URL to `http://<your-ha-ip>:8133/audiobookshelf/`
4. Choose an icon (e.g., `mdi:book-open-page-variant`)
5. Click **Create**

For external access (via DuckDNS/Nginx Proxy Manager), use your external URL with the port forwarded.

## Reverse Proxy Setup

To place Audiobookshelf behind a reverse proxy (e.g., Nginx Proxy Manager, Caddy, Traefik):

1. Forward the proxy to the add-on's port: `http://<ha-ip>:8133`
2. Keep `router_base_path` set to `/audiobookshelf` (default).
3. If you want to serve from a different subpath, change `router_base_path` to match.
4. For Nginx with `X-Accel-Redirect` optimized media delivery, enable `use_x_accel`.

## OpenID Connect (SSO)

Audiobookshelf supports OpenID Connect for single sign-on. Configure it through the web UI at **Settings > Authentication**, or set the relevant environment variables via the `env_vars` option:

```yaml
env_vars:
  - name: AUTH_OPENID_ISSUER_URL
    value: https://your-idp.example.com/.well-known/openid-configuration
  - name: AUTH_OPENID_CLIENT_ID
    value: your-client-id
  - name: AUTH_OPENID_CLIENT_SECRET
    value: your-client-secret
```

Refer to the [upstream authentication documentation](https://www.audiobookshelf.org/docs#authentication) for all available OpenID settings.

## Troubleshooting

### Add-on won't start

- Check the add-on **Log** tab for errors.
- Ensure `config.yaml` schema validation passed (no red banner on the Configuration page).
- If you see "Permission denied" errors, verify the add-on has access to the configured media paths.

### Media files not found

- Use the **container path** in the Audiobookshelf library settings (e.g., `/media/audiobooks`), not the host path.
- Ensure the files are in the correct HA directory (media, share).
- For network-mounted libraries, the file watcher may not work reliably. Use periodic scans in the library settings as a fallback.

### Multiple books merged into one

- Each audiobook must be in its own subfolder. If all files are in the root library folder, they will be treated as separate books — but nested folders without proper structure may merge unexpectedly.

### Password reset

An admin can reset any user's password from **Settings > Users**. If you need to reset the root user password, stop the add-on, then set the `pash` column to `NULL` in the SQLite `users` table inside `/data/config/`.
